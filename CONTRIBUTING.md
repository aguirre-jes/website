# Guía para colaborar en el proyecto

¡Gracias por tu interés en contribuir a este proyecto open source!

## Estándar de colaboración

Este proyecto sigue las mejores prácticas de la comunidad open source para proyectos de reconocimiento. Todas las contribuciones deben cumplir con el Código de Conducta y las políticas de colaboración descritas aquí.

## Cómo colaborar

1. **Fork del repositorio**: Haz un fork del repositorio para trabajar en tus cambios de manera independiente.
2. **Crear un issue**: Antes de enviar un Pull Request (PR), abre un issue para discutir tu propuesta o reportar un bug. Utiliza las plantillas de issues disponibles para facilitar la comunicación.
3. **Trabaja en tu fork**: Realiza los cambios en una rama descriptiva en tu fork.
4. **Pull Request**: Envía un PR desde tu fork. Utiliza la plantilla de PR para describir tus cambios.
5. **DCO (Developer Certificate of Origin)**: Todos los PRs deben incluir la firma DCO para validar que eres el autor de los cambios.

## Proceso de fork y Pull Request

Para contribuir, sigue estos pasos:

1. Haz un fork del repositorio en tu cuenta de GitHub.
2. Crea una rama descriptiva para tus cambios (por ejemplo, `fix-issue-123` o `feature-nueva-funcionalidad`).
3. Realiza los cambios y asegúrate de que cumplen con las guías del proyecto.
4. Envía un Pull Request desde tu rama hacia el repositorio principal. Utiliza la plantilla de PR y relaciona el issue correspondiente.
5. Espera la revisión de los mantenedores y responde a los comentarios si es necesario.

## Validación DCO en Pull Requests

Todos los Pull Requests deben estar firmados con el Developer Certificate of Origin (DCO). Esto certifica que eres el autor de los cambios y que los aportas bajo la licencia del proyecto.

Para firmar tu commit, utiliza:

    git commit -s -m "Mensaje del commit"

Si tu PR no está firmado, el sistema lo marcará como inválido y deberás corregirlo antes de que sea aceptado.

## Plantillas

- Existen plantillas para issues y PRs que debes utilizar para facilitar la revisión y seguimiento.

## Validación DCO

El proyecto valida automáticamente la firma DCO en cada PR. Si tu PR no está firmado, no podrá ser aceptado.
